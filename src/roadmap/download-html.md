The PR flow requires a GitHub token with write access to the target repo. That's a real friction point for users who want to share their bible without granting the app those scopes. Download as HTML is the no-auth alternative: a single file, fully self-contained, that can be dropped into an intranet, a Notion page, an S3 bucket, or a team wiki.

This is not a lesser version of the PR flow — it's a different use case entirely, and for many teams it's the better one. Both options live side by side in the toolbar.

## What it looks like

A "Download HTML →" button appears in the toolbar next to "Push to GitHub →". Clicking it triggers a brief generation step (progress indicator inline, same pattern as the Push button) and then downloads a single `.html` file named `<repo-name>-bible.html`.

The downloaded file opens in any browser with no server required. It looks identical to the in-app dashboard: dark IDE aesthetic, full sidebar with all files, clickable navigation, rendered Mermaid diagrams, syntax-highlighted code. All fonts, styles, and scripts are inlined — zero external dependencies.

## Key details

- The HTML bundle inlines everything: styles, fonts (as base64 data URIs), the Mermaid JS bundle, the highlight.js bundle, all generated markdown rendered to HTML.
- Navigation is JavaScript-driven inside the file — clicking a file in the sidebar shows its writeup without a network request.
- The file is named `<owner>-<repo>-bible.html`.
- The architecture overview is the default view on open.
- Generation happens server-side: a new `exportBibleHtml` method assembles the bundle and returns it as a base64-encoded string. The frontend decodes it and triggers a download via a programmatic `<a>` click.
- The exported file includes a small footer: "Generated by Codebase Bible on <date>." with a link back to the app.

~~~
New method: `exportBibleHtml({ repositoryId })`. Fetches all generated docs for the repo (same query as `getRepositoryDetail`). Renders each markdown document to HTML using a server-side markdown parser. Assembles a single HTML string with all content, navigation JS, and inlined assets. Returns `{ html: string, filename: string }` (html is the full document as a string; the frontend creates a Blob and triggers download).

Assets to inline: JetBrains Mono and Inter as base64 data URIs (subset to the characters actually used, or use system-font fallbacks to keep file size reasonable). Mermaid and highlight.js can be inlined from their CDN bundles. Total file size target: under 2MB for a typical repo.

Frontend: the Download button follows the same `pushing` boolean pattern as the PR button. On success, construct a Blob from the returned HTML string, create an object URL, programmatically click a hidden `<a download>` element, then revoke the URL.
~~~
