### 4. Internal Logic Walkthrough
The file's core logic ensures a single Prisma Client instance is created and reused. It first checks if a client already exists on the global object (for development hot reloading), and if not, creates a new instance with environment-specific logging.

```typescript
const globalForPrisma = global as unknown as { prisma: PrismaClient };

export const prisma =
  globalForPrisma.prisma ||
  new PrismaClient({
    log: process.env.NODE_ENV === 'development' ? ['error', 'warn'] : ['error'],
  });
```
- **Why**: In development, hot reloading can cause multiple Prisma Client instances, exhausting database connections. Attaching to `global` prevents this. The logging configuration reduces verbosity in production by only logging errors.

```typescript
if (process.env.NODE_ENV !== 'production') {
  globalForPrisma.prisma = prisma;
}
```
- **Why**: This persists the client instance on the global object only in non-production environments, ensuring hot reloads reuse the same instance. In production, a new instance is created per process (e.g., in serverless deployments).
